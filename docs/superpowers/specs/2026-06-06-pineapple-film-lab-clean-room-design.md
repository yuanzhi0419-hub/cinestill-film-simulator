# 凤梨罐头 FILM LAB 洁净室重写设计

## 1. 项目目标

凤梨罐头 FILM LAB 是面向摄影爱好者的本地图像处理应用。用户可以在浏览器中处理 JPG、PNG 和常见 RAW 照片，使用自研预设、基础色调、光晕、颗粒、暗角和用户自带 LUT，并批量导出结果。

项目采用全新目录和 Git 历史，以 MIT 许可证发布。实现不得复制旧项目的源码、页面、文案、测试、LUT、XMP 或样片。

## 2. 洁净室边界

- 只依据已经确认的产品需求重新设计和实现。
- 不打开旧项目源码作为实现参考。
- 不复制旧项目中的算法结构、函数命名、接口实现或页面布局代码。
- 不包含旧项目及第三方来源不明确的 LUT、XMP 和样片。
- 内置预设仅保存本项目独立定义的参数组合。
- 用户可以导入自己拥有使用权的 `.cube` LUT。
- 新仓库明确记录独立实现原则、许可证和依赖来源。

## 3. 用户与运行方式

- 目标用户：摄影爱好者。
- 运行方式：本地浏览器应用。
- 隐私原则：照片和处理结果不上传网络。
- 历史记录：仅保留当前进程会话，服务关闭后清理。
- 首轮目标平台：macOS、Windows 和 Linux。

## 4. 技术架构

### 4.1 后端

- Python 3.11 或更高版本。
- Flask 提供本地页面和 JSON/文件接口。
- Pillow、NumPy 和 OpenCV 构成独立图像处理管线。
- rawpy 解码支持的 RAW 文件。
- 进程内任务队列负责批量任务、进度、取消和失败隔离。
- 每次启动创建独立临时会话目录，正常退出时清理。

不引入 Redis、Celery、数据库或外部云服务。

### 4.2 前端

- 使用原生 HTML、CSS 和 JavaScript。
- 不引入前端框架和构建链。
- 使用暗房轻量工作台布局。
- 参数变化采用防抖处理，只请求低分辨率预览。
- 任务状态通过轮询本地接口更新。

### 4.3 模块边界

- `app/`: Flask 应用创建、配置和路由。
- `processing/`: 解码、调整步骤、预设和输出。
- `jobs/`: 任务模型、进程内队列和生命周期。
- `storage/`: 临时会话文件、清理和安全路径管理。
- `web/`: 页面模板、样式和浏览器交互。
- `tests/`: 单元、接口和任务队列测试。

模块通过明确的数据对象交互，不直接读取其他模块的内部状态。

## 5. 功能范围

### 5.1 输入

- 支持一次上传一张或多张照片。
- 支持 JPG、PNG 和 rawpy 可解码的常见 RAW 格式。
- 验证扩展名、内容类型、文件大小和实际解码结果。
- 修正 EXIF 方向。
- 透明 PNG 在预览和 JPEG 导出时使用明确的背景合成策略。

### 5.2 自研预设

首版提供以下独立参数预设：

- 夜行
- 清晨
- 自然负片
- 柔雾
- 黑白纪实

预设不是第三方胶片产品的复刻，不使用第三方商标命名。

### 5.3 参数

- 预设强度
- 曝光
- 对比度
- 高光
- 阴影
- 色温
- 饱和度
- 光晕
- 颗粒
- 暗角

所有参数有明确范围、默认值和重置行为。

### 5.4 对比和批量处理

- 原图查看。
- 效果图查看。
- 可拖动分割对比。
- 当前图片参数可一键应用到全部图片。
- 队列显示等待、解码、处理、完成、失败和已取消状态。
- 单项失败不阻塞其他任务。
- 支持单项重试和移除。

### 5.5 导出

- 单张导出 JPEG。
- 批量导出 ZIP。
- 原始分辨率处理在后台队列执行。
- 文件名包含原文件名和本项目后缀，避免覆盖原图。

## 6. 图像处理管线

固定处理顺序：

1. 解码
2. EXIF 方向校正
3. 基础色调
4. 色温与饱和度
5. 高光光晕
6. 颗粒
7. 暗角
8. 色域收敛与输出

预览使用受限尺寸和较快参数；导出使用原始尺寸。两条路径共享同一处理定义，避免预览与导出结果出现逻辑差异。

光晕、颗粒和预设算法必须从数学定义重新实现，不参考旧项目实现。

## 7. 数据流

1. 上传文件写入当前临时会话目录。
2. 后端验证并生成轻量缩略图。
3. 前端选择第一张有效图片并展示预览。
4. 参数变化经过防抖后请求低分辨率预览。
5. 后端返回最新参数版本对应的预览；过期结果被前端忽略。
6. 用户发起导出后创建原始分辨率任务。
7. 前端轮询队列状态。
8. 完成后提供 JPEG 或 ZIP 下载。
9. 服务关闭后清理整个会话目录。

## 8. 界面设计

### 8.1 桌面布局

- 顶部：项目名称、添加照片、对比模式和导出。
- 左侧：自研预设和用户 LUT 导入。
- 中间：大尺寸照片画布。
- 右侧：参数滑杆、数值输入和重置。
- 底部：批量缩略图队列、状态、进度和移除。

### 8.2 移动端布局

- 预览置顶。
- 预设和参数放入底部标签页。
- 队列横向滚动。
- 主要按钮保持可见且不遮挡内容。

### 8.3 视觉风格

- 炭黑背景提供稳定的照片观看环境。
- 黄色表示选中状态。
- 珊瑚红表示主要操作。
- 青色表示参数值和处理反馈。
- 不使用装饰性渐变、光球、大圆角卡片或营销页结构。
- 控件使用熟悉图标，并为不直观的图标提供工具提示。

### 8.4 交互规则

- 上传后自动选中第一张有效照片。
- 默认只修改当前照片。
- 可将当前参数应用到全部照片。
- 导出期间可继续编辑其他照片。
- 错误显示在对应队列项和页面状态区域，不使用浏览器弹窗。
- 页面明确显示所有处理均在本机完成。

## 9. 错误处理

- 无效文件在上传阶段返回具体原因。
- 单张解码失败不会影响同批其他文件。
- 参数越界返回结构化错误。
- 预览请求失败保留上一张有效预览。
- 任务失败记录错误摘要并允许重试。
- 找不到或无权读取临时文件时拒绝访问，不回退到任意本地路径。
- 服务关闭时尽力取消任务并清理临时目录。

## 10. 测试策略

### 10.1 单元测试

- 每个图像处理步骤。
- 预设参数和处理顺序。
- 参数校验。
- 临时文件路径和清理。
- 任务状态转换、取消和失败隔离。

### 10.2 接口测试

- JPG 和 PNG 上传。
- 透明 PNG。
- EXIF 方向。
- 无效内容和超限文件。
- RAW 解码使用小型夹具或 mock。
- 预览版本和过期结果处理。
- 单张导出、批量 ZIP、取消和重试。

### 10.3 界面验收

- 桌面和移动端无溢出、遮挡和布局跳动。
- 上传、预览、调整、批量处理、失败重试和下载流程完整。
- 连续拖动滑杆不会产生过量请求。
- 大图和批量任务期间页面保持响应。
- 浏览器控制台没有相关错误。

- 自动化测试全部通过。

- 发布前执行源文件和资源来源审计。

## 12. 非目标

- 不提供云端账户、同步或托管。
- 不保存跨会话照片历史。
- 不复刻特定商业胶片品牌。
- 不提供未经授权的 LUT 或预设下载。
- 不实现插件系统、数据库或分布式任务队列。
- 首版不追求专业色彩管理、显示器校准或打印软打样。
